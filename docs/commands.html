CLI Commands Reference

Master the HydrivaX toolchain. All commands are designed to be intuitive and beautiful.

hx-deploy

The package and module manager for HydrivaX.

Purpose

Downloads, configures, and initializes software modules securely. It resolves dependencies and configures systemd services automatically.

Syntax

hx-deploy <action> [module_name]

Parameters

  • install : Installs the specified module. If no action is provided, install is assumed.
  • remove : Uninstalls a module and its dependencies.
  • list : Lists all available and currently installed modules.
  • search : Searches the module catalog for a keyword.

Example Use Cases

Deploying a web server stack quickly, setting up a VPN, or removing unneeded software to keep the system footprint small.

root@hydrivax:~# hx-deploy docker

Example Output

[*] Fetching module 'docker'...
[*] Installing dependencies: containerd, runc... Done.
[*] Docker service started and enabled.

hx-status

System telemetry and health dashboard.

Purpose

Displays a beautiful, colorized overview of system resource utilization (CPU, RAM, Disk) and lists currently running HydrivaX modules.

Syntax

hx-status

Example Use Cases

Quickly checking if your container is running out of memory, or verifying that a recently deployed service is running successfully.

hx-info

Display OS release and architecture details.

Purpose

Provides a quick summary of the HydrivaX version, the underlying base system, the active edition (Server, Desktop, LXC), and footprint size.

Syntax

hx-info

hx-update

System and module updater.

Purpose

Fetches the latest security patches for the base system and updates the definitions for the `hx-deploy` module catalog.

Syntax

hx-update [--full]

hx-install

Launch the bare-metal/VM installer.

Purpose

When booted from the Live ISO, this command launches the interactive installer to partition the disk and write the HydrivaX OS permanently to the hardware.

Syntax

hx-install
