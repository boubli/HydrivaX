Install

Boot HydrivaX OS from ISO or deploy the LXC template on Proxmox.

Boot from ISO

HydrivaX OS v1.2 — VirtualBox, Proxmox VMs, VMware, mini PCs, and physical hardware.

💻 Option A: Virtual Machines

Run HydrivaX OS inside a hypervisor (Proxmox VE VM, VirtualBox, or VMware). Perfect for testing, development, and clean sandboxed hosting environments.

  • Upload ISO to Proxmox or VM datastore
  • Create VM with 1 CPU, 1-2 GB RAM, SATA/VirtIO
  • Attach ISO as virtual optical drive
  • Boot and access via noVNC / VGA Console

🔌 Option B: Physical Hardware

Install HydrivaX OS directly on bare metal (Mini PCs, Intel NUCs, old desktops, or dedicated rack servers) to utilize 100% of physical hardware power.

  • Flash ISO to USB using Rufus, BalenaEtcher, or Ventoy
  • Insert USB into the target machine
  • Access BIOS/Boot Menu (usually F12, F11, or Esc)
  • Boot from USB (UEFI & Legacy BIOS are supported)
1

Download the ISO & Verify

Grab HydrivaX-v1.2-Server-amd64.iso from the Download page. To guarantee a clean, uncorrupted image, verify the SHA256 checksum in your terminal:

# Windows (PowerShell) Get-FileHash HydrivaX-v1.2-Server-amd64.iso -Algorithm SHA256 # Linux / macOS (Terminal) sha256sum HydrivaX-v1.2-Server-amd64.iso

Compare the output hash with the checksums listed on the downloads page.

2

Prepare Boot Medium

Choose the method below that matches your deployment target:

  • For Proxmox VE (VM): Upload the ISO via the Proxmox Web UI (Local storage → ISO Images → Upload) or copy it directly using SCP:
    $ scp HydrivaX-v1.2-Server-amd64.iso root@PROXMOX_IP:/var/lib/vz/template/iso/
  • For VirtualBox & VMware: Save the ISO in a permanent folder on your host machine.
  • For Physical Hardware (USB Flashing): Flash the ISO to a USB flash drive (8GB+ recommended).
    Tools: Use **Rufus** (Windows - select ISO mode, re-flash in DD mode if boot fails), **BalenaEtcher** (Cross-platform), or **Ventoy** (simply drag and drop the ISO onto the Ventoy drive).
3

Platform Configuration

Set up your environment settings according to the platform you are deploying on:

Proxmox VE (KVM VM):

Create a VM. Allocate 1–2 vCPUs (Host type recommended), 1–2 GB RAM, and 10–20 GB storage. Attach the ISO to the CD/DVD drive.
Important: To avoid black screen issues in noVNC, add a Serial terminal 0 in the hardware settings and set display to Serial terminal 0.

VirtualBox:

Create a new VM, select type Linux and version Debian (64-bit). Allocate 1 CPU and 1–2 GB RAM. Mount the ISO under Storage settings.
Tip: Ensure Enable EFI is unchecked in System settings, and set the Graphics Controller to VBoxSVGA or VMSVGA under Display.

VMware Workstation / Player:

Create a VM with typical configuration, select installer disc image (ISO), and point to the HydrivaX ISO. Choose Guest OS Linux and version Debian 12 64-bit. Allocate 1 CPU, 1-2 GB RAM, and set Network Connection to NAT or Bridged.

Physical Hardware (Bare Metal / Mini PC):

Insert the flashed USB drive. Power on the machine and immediately press the Boot Menu hotkey (F12, F11, F8, F9, or Esc depending on manufacturer). Select the USB boot option.
Tip: Enter BIOS (Del or F2) and **disable Secure Boot**. You may also need to enable Legacy Support/CSM if using older legacy boot modes.

4

Boot & Log in

Start the VM or physical machine. At the GRUB boot menu, choose your console option:

  • VGA Text: Standard console output (best for VirtualBox, VMware, and physical monitors).
  • Serial Console: Redirects boot output to serial ttyS0 (best for headless servers and Proxmox noVNC serial console).
  • Fail-safe: Safe mode using basic drivers for older or difficult hardware.

Once loaded, log in with the live system credentials:

user live pass hydrivax
5

First commands

Your live server is now up! Run information and status commands, or list the deployable modules:

$ hx-info $ hx-status $ hx-deploy list

Proxmox LXC template

Deploy HydrivaX as a lightweight container on Proxmox VE.

1

Download the template

Grab HydrivaX-v2.0-LXC.tar.zst from the Releases page.

2

Upload to Proxmox

Copy the archive to your Proxmox template cache directory.

$ scp HydrivaX-v2.0-LXC.tar.zst root@PROXMOX_IP:/var/lib/vz/template/cache/
3

Create a container

Open the Proxmox web UI, go to your node → CT Templates, and create a new LXC container from the HydrivaX template. Configure CPU, RAM, and storage as needed.

4

First commands

SSH into your new container and explore the CLI toolkit.

$ hx-info $ hx-deploy list $ hx-status $ hx-deploy docker

What to expect

On first login you'll see the HydrivaX MOTD banner. The system is intentionally minimal — install only what you need.

root@hydrivax:~
