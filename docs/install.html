Install

Boot HydrivaX OS from ISO or deploy the LXC template on Proxmox.

Boot from ISO

HydrivaX OS v1.2 — VirtualBox, Proxmox VMs, mini PCs, and physical hardware.

💻 Option A: Virtual Machines

Run HydrivaX OS inside a hypervisor (Proxmox VE VM, VirtualBox, or VMware). Perfect for testing, development, and clean sandboxed hosting environments.

  • Upload ISO to Proxmox or VM datastore
  • Create VM with 1 CPU, 1-2 GB RAM, SATA/VirtIO
  • Attach ISO as virtual optical drive
  • Boot and access via noVNC / VGA Console

🔌 Option B: Physical Hardware

Install HydrivaX OS directly on bare metal (Mini PCs, Intel NUCs, old desktops, or dedicated rack servers) to utilize 100% of physical hardware power.

  • Flash ISO to USB using Rufus, BalenaEtcher, or Ventoy
  • Insert USB into the target machine
  • Access BIOS/Boot Menu (usually F12, F11, or Esc)
  • Boot from USB (UEFI & Legacy BIOS are supported)
1

Download the ISO

Grab HydrivaX-v1.2-Server-amd64.iso from the Download page. Ensure you verify the SHA256 checksum to guarantee a clean and corruption-free image.

2

Prepare Boot Medium

For VMs (Proxmox VE): Upload the ISO to your ISO library (e.g., local storage) via the Proxmox Web UI, or use SCP:

$ scp HydrivaX-v1.2-Server-amd64.iso root@PROXMOX_IP:/var/lib/vz/template/iso/

For Physical Hardware: Write the ISO to a USB flash drive (8GB+ recommended) using Rufus (use DD image mode if prompted) or BalenaEtcher.

3

Configure Settings & Boot

For Virtual Machines: Create a new KVM VM. Assign 1–2 vCPUs, 1–2 GB RAM, and a VirtIO Network interface. Under hardware, add a CD/DVD drive, mount the ISO, and set it first in the Boot Order settings.

Pro Tip: In Proxmox, add a Serial terminal 0 and set display to Serial terminal 0 if you get a black screen in noVNC.

For Physical Hardware: Plug the USB drive into your target machine. Power it on and immediately press the Boot Menu key (often F12, F11, F8, or Esc). Select your USB drive from the boot options.

Pro Tip: Enter BIOS settings and disable Secure Boot. Custom live images can sometimes be blocked by UEFI secure boot keys.

4

Log in

At the GRUB boot menu, choose one of the following boot entries:

  • VGA Text: Standard console output (recommended for VirtualBox and local monitors).
  • Serial Console: Directs boot logs to serial ttyS0 console (recommended for headless Proxmox VM consoles).
  • Fail-safe: Safe mode using basic drivers for older or difficult hardware.

Once loaded, log in with the live system credentials:

user live pass hydrivax
5

First commands

Your live server is now up! Run information and status commands, or list the deployable modules:

$ hx-info $ hx-status $ hx-deploy list

Proxmox LXC template

Deploy HydrivaX as a lightweight container on Proxmox VE.

1

Download the template

Grab HydrivaX-v2.0-LXC.tar.zst from the Releases page.

2

Upload to Proxmox

Copy the archive to your Proxmox template cache directory.

$ scp HydrivaX-v2.0-LXC.tar.zst root@PROXMOX_IP:/var/lib/vz/template/cache/
3

Create a container

Open the Proxmox web UI, go to your node → CT Templates, and create a new LXC container from the HydrivaX template. Configure CPU, RAM, and storage as needed.

4

First commands

SSH into your new container and explore the CLI toolkit.

$ hx-info $ hx-deploy list $ hx-status $ hx-deploy docker

What to expect

On first login you'll see the HydrivaX MOTD banner. The system is intentionally minimal — install only what you need.

root@hydrivax:~
