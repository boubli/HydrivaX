Deploying on Proxmox VE (LXC)

HydrivaX was originally built for Proxmox LXC. This is the fastest and most efficient deployment method.

Requirements

  • Proxmox VE 7.0 or newer
  • Internet access on the PVE host
  • At least 512MB RAM and 2GB disk space for the container

Step-by-Step Instructions

  1. Log into your Proxmox VE node shell via SSH or the web UI.
  2. Execute the Autopilot installer script:
    root@pve:~# bash -c "$(curl -fsSL https://boubli.github.io/HydrivaX/install-lxc.sh)"
  3. Follow the interactive prompts to assign a Container ID, Hostname, and Root Password.
  4. Wait for the template to download and extract (approx. 15 seconds).
  5. Start the container from the Proxmox UI or via `pct start <vmid>`.

Expected Results

You should have a running LXC container using minimal resources (~40MB RAM). You can now log in using the password you created and run hx-status.

Troubleshooting

Container fails to start: Check if you have nesting enabled. The Autopilot script enables it by default, but if you modified it, go to Options -> Features -> Enable Nesting in Proxmox.
No internet inside container: Ensure your Proxmox bridge (`vmbr0`) is configured correctly with DHCP or manually assign a static IP via the Proxmox Network tab for the container.

Deploying as a Virtual Machine

Install HydrivaX Server ISO on VMware, VirtualBox, or KVM.

Requirements

  • Hypervisor (VirtualBox, VMware ESXi/Workstation, KVM)
  • HydrivaX Server ISO
  • At least 1GB RAM and 8GB virtual disk

Step-by-Step Instructions

  1. Download the HydrivaX Server ISO from the downloads page.
  2. Create a new VM in your hypervisor. Choose "Linux 64-bit" or "Debian 12 64-bit" as the guest OS type.
  3. Attach the downloaded ISO to the virtual CD/DVD drive.
  4. Boot the VM. You will be greeted by the live environment.
  5. Log in with user root (no password required in the live environment).
  6. Run the installation command to write HydrivaX to disk:
    root@hydrivax:~# hx install
  7. Follow the on-screen installer prompts.
  8. Reboot and remove the ISO from the virtual drive.

Expected Results

The VM will boot directly from the virtual hard drive into the HydrivaX OS prompt.

Troubleshooting

Installer cannot find disk: Ensure your hypervisor is presenting the disk as SATA or VirtIO, not IDE or SCSI, as the minimal kernel strips some older legacy drivers.

Deploying on Bare Metal

Install HydrivaX directly onto physical hardware like an Intel NUC, Mini PC, or standard server.

Requirements

  • USB Flash Drive (at least 4GB)
  • BalenaEtcher or Rufus
  • Physical server/PC with UEFI boot support

Step-by-Step Instructions

  1. Download the HydrivaX Server ISO.
  2. Flash the ISO to your USB drive using BalenaEtcher or Rufus (choose DD mode if prompted).
  3. Insert the USB drive into your physical machine and boot from it via the BIOS/UEFI boot menu.
  4. Once the live system boots, connect an Ethernet cable.
  5. Run the installer:
    root@hydrivax:~# hx install
  6. Select your primary hard drive during the partition step.
  7. Once complete, remove the USB drive and reboot.

Expected Results

The physical machine will boot into HydrivaX. Network drivers for common Intel/Realtek NICs are included.

Troubleshooting

No network connection: If your specific network card requires proprietary firmware not included in the stripped rootfs, you can deploy the firmware module via hx-deploy linux-firmware-nonfree while temporarily tethered via USB to your phone for internet.

Deploying on a Cloud VPS

Install HydrivaX on DigitalOcean, Hetzner, Linode, or AWS.

Requirements

  • A VPS provider that supports custom ISO mounting or Rescue Mode.

Step-by-Step Instructions

  1. Upload the HydrivaX Server ISO to your cloud provider's custom image library.
  2. Deploy a new instance using the custom ISO.
  3. Open the web console provided by your cloud host.
  4. Run the installation script:
    root@hydrivax:~# hx install
  5. Configure the network manually in the installer if DHCP is not provided by your VPS.
  6. Detach the ISO and reboot the instance.

Expected Results

Your VPS will be accessible via SSH (if you install the OpenSSH module or it was selected during setup).

Troubleshooting

SSH connection refused: HydrivaX is headless and minimal. You must ensure you deploy the ssh server module: hx-deploy openssh-server.
