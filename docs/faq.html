Frequently Asked Questions

Everything you need to know about HydrivaX.

Why use HydrivaX?

HydrivaX bridges the gap between full-fat operating systems like Debian/Ubuntu and highly minimal environments like Alpine. It provides a tiny footprint (~346MB disk, 40MB RAM) while retaining glibc compatibility, allowing you to run any standard Linux software. Its built-in module catalog means you never have to memorize complex installation scripts again.

How is it different from Debian?

While HydrivaX is built on Debian's core packages, we remove thousands of non-essential files: kernel modules, man pages, locales, bootloaders, and firmware. We then layer our custom hx-* toolchain on top, transforming it from a general-purpose OS into a specialized, modular appliance for self-hosting.

How much RAM is required?

HydrivaX idles at just ~40MB of RAM. A container provisioned with 256MB or 512MB of RAM is more than enough to run the base OS and several lightweight services. For heavier modules like Nextcloud or a large database, we recommend allocating 1GB to 2GB.

Can I run Docker?

Absolutely. Docker is a first-class citizen. Simply run hx-deploy docker. If you are running HydrivaX inside a Proxmox LXC container, be sure you have "Nesting" and "keyctl" enabled in your container's Features/Options tab so Docker can run properly.

Can I use it on Proxmox?

Yes, Proxmox VE is our primary and most optimized platform. HydrivaX was built specifically to solve the headaches of LXC containers on Proxmox. You can deploy it instantly via our Autopilot script on the Quick Start page.

How are updates handled?

Updates are dead simple. Just type hx-update. This will automatically update your module catalog, apply the latest security patches from Debian upstream, and clean up residual packages.

Is it production ready?

HydrivaX v2.0 is considered stable and production-ready for homelabbers, self-hosters, and SME deployments. Because it relies on the rock-solid foundation of Debian 12 and systemd, the underlying OS is extremely resilient. However, always ensure you have a backup strategy (e.g., using the restic module) before deploying critical data.
